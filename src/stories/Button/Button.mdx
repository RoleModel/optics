# Button

Button classes can be used on `button` or `a` html elements. They provide consistent and composable styling that should address most applications basic needs.

## Default

`.btn` Provides a default outlined black text button. This would be commonly used as a cancel button in a form.

## Outline

`.btn--outline` This is a modifier which can be used with all button classes. It provides an outlined button that adapts it's color to all the btn variations.

## No Border

`.btn--no-border` This is a modifier which can be used with all button classes. It provides a button with no border, but the same sizing and adapts it's color to all the btn variations.

## Disabled

`.btn--disabled` Should be used on `a` elements that need to be disabled. `button` elements can use the `disabled` attribute as well as this class. This makes the button opaque and changes the cursor to the `not allowed` icon.

`.btn--pill` Modifies the shape of any other buttons by adding a radius to the left and right.

## Icon

`.btn--icon` Modifies the shape of any other button class by adding a full radius to the button. This should only be used with an icon as the button text. You can also combine it with `.btn--no-border` for a borderless icon button.

## Size

`.btn--small`, `.btn--medium`, `.btn--large` (with medium being the default) modify the size of any other button class by changing the font and height to be smaller or larger.

## Mobile Buttons

All button classes, regardless of the currently applied size modifier, will automatically adjust to the large button size on mobile screens ($breakpoint-small ~ 768px).
This is to ensure a large enough tap area on smaller devices and screens.

See https://uxmovement.com/mobile/optimal-size-and-spacing-for-mobile-buttons/

## Button API

Size and Font button styles are built on css variables scoped to the button.
Buttons have various states that are styled as well.

Here are the variables and states used

<!-- prettier-ignore-start -->
```css
// Variable API
--_op-btn-height-small:
--_op-btn-height-medium:
--_op-btn-height-large:

--_op-btn-font-small:
--_op-btn-font-medium:
--_op-btn-font-large:

// Different states
.btn {} // Default behavior
.btn:not(:disabled, .btn--disabled):hover {} // Hover behavior
.btn.btn--no-border {} // No Border Modifier
.btn.btn--no-border:not(:disabled, .btn--disabled):hover {} // Hovered No Border Modifier
.btn.btn--outline {} // Outline Modifier
.btn.btn--outline:not(:disabled, .btn--disabled):hover {} // Hovered No Border Modifier
```
<!-- prettier-ignore-end -->

## Overriding Button styles

The button classes are built on a sass placeholder selector https://sass-lang.com/documentation/style-rules/placeholder-selectors

This allows multiple button classes to share the same behavior. You can modify all buttons behavior by overriding the `%btn-global` placeholder selector and setting any properties:

```css
%btn-global {
  border-radius: 0;
}
```

If you want to override how the size modifier behaves, you can use API described above to change which height and font each size uses.

<!-- prettier-ignore-start -->
```css
%btn-global {
  --_op-btn-height-small:  var(--op-space-2x-large);
  --_op-btn-height-medium: var(--op-space-3x-large);
  --_op-btn-height-large:  var(--op-space-4x-large);

  --_op-btn-font-small:  var(--op-font-3x-large);
  --_op-btn-font-medium: var(--op-font-4x-large);
  --_op-btn-font-large:  var(--op-font-5x-large);
}
```
<!-- prettier-ignore-end -->

If you need to override the color of a particular button style, you can open the respective class and change the `background-color`, `color`, and `box-shadow` properties to change the look.

```css
// This will only affect the default button, but not primary, secondary, etc.
.btn {
  background-color: red;
  color: white;
  box-shadow: red;
}
```

## Custom Button

Your application may need a custom button. To add one, just follow this template:

```css
.btn-{name} {
  @extend %btn-global;

  background-color:
  color:
  box-shadow:

  &:not(:disabled, .btn--disabled):hover {
    background-color:
    color:
    box-shadow:
  }

  &.btn--no-border {
    background-color:
    color:
    box-shadow:

    &:not(:disabled, .btn--disabled):hover {
      background-color:
      color:
      box-shadow:
    }
  }

  &.btn--outline {
    background-color:
    color:
    box-shadow:

    &:not(:disabled, .btn--disabled):hover {
      background-color:
      color:
      box-shadow:
    }
  }
}
```

```css
.btn-notice {
  @extend %btn-global;

  background-color: var(--op-color-alerts-notice-base);
  color: var(--op-color-alerts-notice-on-base);
  box-shadow: var(--op-border-all) var(--op-color-alerts-notice-base);

  &:not(:disabled, .btn--disabled):hover {
    background-color: var(--op-color-alerts-notice-plus-one);
    color: var(--op-color-alerts-notice-on-plus-one);
    box-shadow: var(--op-border-all) var(--op-color-alerts-notice-plus-one);
  }

  &.btn--no-border {
    background-color: transparent;
    color: var(--op-color-alerts-notice-base);
    box-shadow: none;

    &:not(:disabled, .btn--disabled):hover {
      background-color: transparent;
      color: var(--op-color-alerts-notice-plus-one);
      box-shadow: none;
    }
  }

  &.btn--outline {
    background-color: var(--op-color-alerts-notice-plus-seven);
    color: var(--op-color-alerts-notice-on-plus-seven);
    box-shadow: var(--op-border-all) var(--op-color-alerts-notice-on-plus-seven);

    &:not(:disabled, .btn--disabled):hover {
      background-color: var(--op-color-alerts-notice-plus-four);
      color: var(--op-color-alerts-notice-on-plus-four);
      box-shadow: var(--op-border-all) var(--op-color-alerts-notice-on-plus-seven);
    }
  }
}
```
