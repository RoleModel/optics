# Card

Card classes can be used to denote bordered sections of an application. They provide simple styles to create sections or "cards" for your interface. They can also be used as a starting point for "row" or list styles.

## Card API

The card styles are built on css variables scoped to the card.

Here is the full public API that the card uses.
```scss
// Normal
--_rm-card-background-color
--_rm-card-text-color

// General Properties
--_rm-card-radius
--_rm-card-border
--_rm-card-font-size
--_rm-card-line-height
--_rm-card-padding
```

## Overriding card styles

The card classes are built on a sass placeholder selector https://sass-lang.com/documentation/style-rules/placeholder-selectors

This allows multiple card classes to share the same behavior. You can modify all cards behavior by overriding the `%card-global` placeholder selector and setting any of the **general properties** documented tokens:
```scss
%card-global {
  --_rm-card-font-size: var(--rm-font-x-large);
}
```

If you need to override the color of a particular card style, you can open the respective class and change the **non-general properties** documented above
```scss
// This will only affect the card, but not card-padded
.card {
  --_rm-card-background-color: red;
  --_rm-card-text-color: white;
}
```

## Custom Card

Your application may need a custom card. To add one:
1. Create a class like `.card-redsky`
2. Extend `%card-global`
3. Use the above documented API to define how it looks and feels for different states (e.g. color, radius)

```scss
.card-redsky {
  @extend %card-global;

  // Normal
  --_rm-card-background-color: red;
  --_rm-card-text-color: white;

  // General Properties
  --_rm-card-radius: var(--rm-radius-large);
  --_rm-card-font-size: var(--rm-font-x-large);
}
```
