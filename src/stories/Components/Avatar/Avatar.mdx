# Avatar

Avatar classes can be used on `a` or `img` html elements.They provide consistent and composable styling for application avatars or profile pictures.

When using on an `a` element, the `img` element should be the only child and set the source to the desired image.
When using on an `img` element, the source and classes are set directly on it.

### Selective Imports

Avatar can be used as a standalone component, however, it does have a few dependencies. To see a full dependency list, see [Dependency Graph](?path=/docs/overview-selective-imports--docs#dependencies)

```css
// Depends on
@import '@rolemodel/optics/dist/scss/core/tokens';
@import '@rolemodel/optics/dist/scss/core/base';

// Component
@import '@rolemodel/optics/dist/scss/components/avatar';
```

## Variations

### Default

`.avatar` is the default avatar class.

<Canvas of={AvatarStories.Default} />

### Disabled

`.avatar--disabled` will disable the avatar and give it an opacity.

<Canvas of={AvatarStories.Disabled} />

### Size

`.avatar--small`, `.avatar--medium`, `.avatar--large` (with large being the default) modify the size of any avatar by changing the width and height to be smaller or larger.

<Canvas of={AvatarStories.Small} />

### Without Link

The avatar classes can be used either on an `a` element that wraps an `img` element, or directly on an `img` element.

`a` is a good choice for avatars that may have interaction such as a link to the users profile or settings.
`img` is a good choice if it is purely to display an image or profile picture and does not need to be interactive. Note: the hover and focus outlines will not be visible when using an `img` element directly.

## Avatar API

The size and border styles are built on css variables scoped to the avatar.

Here are the variables that can be customized.

{/* prettier-ignore-start */}
```css
--_op-avatar-border-radius
--_op-avatar-inner-border-color
--_op-avatar-outer-border-color
--_op-avatar-inner-border-width
--_op-avatar-outer-border-width
--_op-avatar-disabled-opacity
--_op-avatar-size-small
--_op-avatar-size-medium
--_op-avatar-size-large
```
{/* prettier-ignore-end */}

## Overriding styles

The avatar classes are built on a [sass placeholder selector](https://sass-lang.com/documentation/style-rules/placeholder-selectors)

This allows multiple classes to share the same behavior. You can modify all avatar behavior by overriding the `%avatar-global` placeholder selector and setting any properties:

```css
%avatar-global {
}
```

## Custom Avatar

Your application may need a custom avatar. To add one, just follow this template:

```css
.avatar-{name} {
  @extend %avatar-global;

  --_op-avatar-border-radius: var(--op-radius-2x-large);
}
```
